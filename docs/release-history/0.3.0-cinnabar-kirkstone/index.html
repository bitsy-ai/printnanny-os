NATS replaces Cloud IoT Core​

Earlier this month, Google Cloud Platform's IoT Core service was discontinued by Google.

PrintNanny had used IoT Core's managed MQTT broker and identify (certificate) management system since launch.

When Google made the announcement 12 days ago, I migrated PrintNanny's event system to use a NATS message broker and NATS Nkeys for authentication/identity management. You'll notice printnanny-mqtt.service has been removed and replaced by printnanny-nats.service.

Don't worry! NATs is able to broker MQTT protocol, which is a staple in the internet-of-things universe.

PrintNanny will continue to support both MQTT and websocket-based events. 🙌

PrintNanny API Docs​

You can now find interactive docs for PrintNanny's API at the following urls:

  • https://printnanny.ai/api/schema/redoc/ (Redoc)
  • https://printnanny.ai/api/schema/swagger-ui/ (Swagger)

PrintNanny's API is defined by an OpenAPI schema, which is used to generate API client code in Rust, Typescript, and Python.
